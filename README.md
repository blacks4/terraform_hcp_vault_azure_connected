# HCP Vault on Azure

This Terraform creates a private HCP Vault cluster and connects it to an Azure VNet through HCP HVN peering. It also creates one Ubuntu VM in Azure to test private connectivity to Vault.

## Files

- `terraform.tf`: required Terraform version, provider versions, and provider configuration for `azurerm`, `azuread`, `external`, and `hcp`.
- `variables.tf`: input variables used by this configuration.
- `hcp.tf`: HCP HVN, HCP Vault cluster, HCP Azure peering request, Azure peering permissions, and HCP route.
- `azure.tf`: Azure resource group, network, NSG, NIC, optional public IP, Ubuntu VM, and VM SSH key data source.
- `outputs.tf`: Vault, HVN, peering, VM IP, SSH command, and private key path outputs.
- `terraform.tfvars.example`: example values for this configuration.
- `scripts/ensure_ssh_key.sh`: helper that creates or reuses the SSH key pair for the VM.

## HCP Resources Created

- `hcp_hvn.this`: Azure HVN using `hvn_id`, `hcp_region`, and `hvn_cidr_block`.
- `hcp_azure_peering_connection.this`: peering request from the HCP HVN to the Azure VNet.
- `data.hcp_azure_peering_connection.active`: waits for the Azure peering connection to become active.
- `hcp_hvn_route.azure_vnet`: HCP route from the HVN to `azure_vnet_cidr`.
- `hcp_vault_cluster.this`: private HCP Vault `standard_small` cluster in the HVN.

The Vault cluster has `public_endpoint = false`, `proxy_endpoint = "DISABLED"`, and `prevent_destroy = true`.

## Azure Resources Created

- `azurerm_resource_group.this`: resource group.
- `azurerm_virtual_network.this`: VNet using `azure_vnet_cidr`.
- `azurerm_subnet.compute`: subnet using `azure_subnet_cidr`.
- `azurerm_network_security_group.compute`: NSG for the VM subnet.
- `azurerm_subnet_network_security_group_association.compute`: attaches the NSG to the subnet.
- `azurerm_public_ip.vm`: optional static public IP when `create_vm_public_ip = true`.
- `azurerm_network_interface.vm`: NIC for the VM.
- `azurerm_linux_virtual_machine.this`: Ubuntu 22.04 VM for connectivity testing.
- VM OS disk: created by the VM resource with `Standard_LRS` storage.
- `azuread_service_principal.hcp_peering`: uses the HCP peering application ID returned by HCP.
- `azurerm_role_definition.hcp_peering`: custom VNet-scoped role for HCP peering.
- `azurerm_role_assignment.hcp_peering`: assigns the custom role to the HCP peering service principal.

## Azure Networking Required for HCP Vault

Azure admins need these pieces for HCP Vault private connectivity:

- An Azure subscription and tenant matching `azure_subscription_id` and `azure_tenant_id`.
- A resource group in `azure_location`.
- A VNet named by `azure_vnet_name` with address space `azure_vnet_cidr`.
- A subnet named by `azure_subnet_name` with prefix `azure_subnet_cidr`.
- No CIDR overlap between `azure_vnet_cidr` and the HCP HVN CIDR, `hvn_cidr_block`.
- An NSG associated to the subnet with outbound TCP `8200` allowed to `hvn_cidr_block`.
- The HCP peering Entra service principal for the application ID returned by `hcp_azure_peering_connection.this.application_id`.
- A custom Azure role scoped to the VNet with these exact actions:
  - `Microsoft.Network/virtualNetworks/peer/action`
  - `Microsoft.Network/virtualNetworks/virtualNetworkPeerings/read`
  - `Microsoft.Network/virtualNetworks/virtualNetworkPeerings/write`
- A role assignment binding that custom role to the HCP peering service principal at the VNet scope.

For the test VM, Azure also needs:

- A NIC in the subnet.
- An Ubuntu 22.04 VM attached to that NIC.
- SSH public key authentication using the key generated by `scripts/ensure_ssh_key.sh`.
- Optional static public IP and inbound TCP `22` from `ssh_allowed_cidrs` when public SSH access is required.

The peering service principal, custom role, and role assignment are the high-permission Azure steps. Without them, HCP cannot activate the private HVN-to-Azure VNet peering connection.

## Usage

```sh
cp terraform.tfvars.example terraform.tfvars
terraform init
terraform plan
terraform apply
```

Useful outputs:

```sh
terraform output vault_private_endpoint_url
terraform output azure_peering_state
terraform output azure_vm_ssh_command
```

Keep `terraform.tfvars`, Terraform state, and generated SSH keys out of version control.
